# M2M 课程设计产品需求文档（PRD）

## 1. 文档信息
- 产品名称：M2M 终端监测与控制教学演示系统
- 文档版本：v1.0
- 日期：2026-04-30

## 2. 产品愿景
构建一个可演示、可评分、可复现的微型 M2M 系统，实现“数据采集 -> 平台决策 -> 远程控制 -> 结果反馈”的完整闭环。

## 3. 核心场景
1. **实时监测**：终端设备持续上报状态，实时展示。
2. **远程控制**：页面下发命令，终端按预期改变。
3. **异常处理**：离线或数据异常触发告警。
4. **结果回溯**：查看关键事件与操作记录。

## 4. 产品范围 (In Scope)
1. 三节点状态展示 (terminal-a, terminal-b, gateway)。
2. 实时数据看板。
3. 控制台操作入口。
4. 告警展示。
5. 事件时间线。

## 5. 信息架构
1. **总览页 (Dashboard)**: 系统状态、在线状态、数据卡片、告警摘要。
2. **控制页 (Control)**: 控制面板、执行结果反馈。
3. **事件页 (Events)**: 操作与状态变更时间线、告警记录。
4. **演示页 (Demo Mode)**: 流程提示、当前步骤状态。

## 6. 验收标准
- 三设备状态实时更新。
- 控制命令触发状态变化且有反馈。
- 告警与恢复均有记录。
- 演示流程流畅。
