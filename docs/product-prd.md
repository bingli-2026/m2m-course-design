# M2M 课程设计产品需求文档（PRD）

## 1. 文档信息

- 产品名称：M2M 终端监测与控制教学演示系统
- 文档版本：v1.0
- 日期：2026-04-30
- 范围：仅定义最终产品形态、用户价值与验收标准，不涉及技术实现方案

## 2. 产品愿景

构建一个可演示、可评分、可复现的微型 M2M 系统，让评审在短时间内直观看到“数据采集 -> 平台决策 -> 远程控制 -> 结果反馈”的完整闭环价值。

## 3. 目标用户与使用场景

### 3.1 目标用户

1. 课程教师/评审老师：关注可展示性、完整性、稳定性、可评分性。
2. 学生团队：关注可操作性、可复现实验流程、答辩表达清晰度。
3. 课堂观众：关注可视化效果与交互反馈是否直观。

### 3.2 核心场景

1. 实时监测场景：终端设备持续上报状态，页面实时展示。
2. 远程控制场景：操作员在页面下发控制命令，终端行为按预期改变。
3. 异常处理场景：设备离线或数据异常时，系统触发告警并提示恢复状态。
4. 结果回溯场景：演示结束后可回看关键事件与操作记录。

## 4. 产品目标与非目标

### 4.1 产品目标

1. 在 5-8 分钟内完成一次标准化闭环演示。
2. 提供可量化指标用于课程评分（时延、成功率、稳定性、恢复能力）。
3. 支持同学在不同电脑环境下按手册复现演示流程。

### 4.2 非目标

1. 不构建通用 IoT 平台（不追求多租户、多协议全覆盖）。
2. 不追求生产级大规模并发能力。
3. 不扩展过多业务模块，优先保证核心闭环可演示。

## 5. 产品范围

### 5.1 In Scope（本期范围）

1. 三节点设备状态展示（terminal-a、terminal-b、gateway）。
2. 实时数据看板（设备状态、核心数据、更新时间）。
3. 控制台操作入口（下发控制命令）。
4. 告警展示（离线、超阈值或状态异常）。
5. 事件时间线（关键操作与状态变化记录）。

### 5.2 Out of Scope（本期不做）

1. 复杂权限系统（多角色精细权限）。
2. 长周期历史报表（按周/月分析）。
3. 商业化部署能力（计费、租户隔离、运维后台）。

## 6. 产品信息架构

1. 总览页（Dashboard）
- 系统状态总览
- 设备在线状态
- 最新数据卡片
- 告警摘要

2. 控制页（Control）
- 设备控制面板
- 指令执行结果反馈

3. 事件页（Events）
- 操作与状态变更时间线
- 告警与恢复记录

4. 演示页（Demo Mode）
- 预设演示流程提示
- 当前步骤状态与通过标记

## 7. 功能需求（按优先级）

### P0（必须有）

1. 设备在线/离线状态可见。
2. 终端数据实时刷新。
3. 页面下发控制命令并有反馈结果。
4. 关键事件可记录并展示。
5. 演示脚本可完整走通一次闭环。

### P1（应有）

1. 异常告警高亮（颜色/标识/提示文案）。
2. 关键指标统计（成功率、响应时间）。
3. 一键重置演示状态（便于多次答辩演练）。

### P2（可选）

1. 大屏模式（适配投影展示）。
2. 演示回放（按时间线快速回放关键节点）。
3. 导出演示结果摘要（图片或文本）。

## 8. 关键用户流程

### 8.1 标准演示流程（5-8 分钟）

1. 展示系统就绪与设备在线状态。
2. 演示终端数据实时上报。
3. 从页面发起控制命令。
4. 展示设备侧状态变化与结果回传。
5. 人为触发一次异常（如设备断开/阈值越界）。
6. 展示告警触发、恢复及事件记录。
7. 总结指标与闭环价值。

### 8.2 异常应急流程

1. 若某设备未上线，允许切换到预备设备或预录演示数据。
2. 若控制指令失败，系统应明确失败原因并支持重试。
3. 若大屏页面异常，保留轻量备份视图用于继续答辩。

## 9. 验收标准（DoD）

### 9.1 功能验收

1. 三设备状态在页面可区分且实时更新。
2. 控制命令可成功触发目标状态变化，且有成功/失败提示。
3. 告警出现与恢复均有可见记录。
4. 演示全流程可在规定时间内完成。

### 9.2 评分指标建议

1. 指令成功率：>= 95%
2. 状态刷新延迟：体感实时，关键操作可在短时间内反馈
3. 连续稳定运行：满足课堂演示时长要求
4. 异常恢复：在可接受时间内恢复到可演示状态

## 10. 交付物清单

1. 可运行演示系统（含默认演示数据/流程）。
2. 演示脚本（主流程 + 应急预案）。
3. 用户手册（安装、启动、演示步骤）。
4. 指标报告（成功率、时延、异常恢复）。
5. 演示视频（标准流程一次 + 异常恢复一次）。

## 11. 里程碑（产品视角）

1. M1：闭环最小可演示版本（P0 全部可用）。
2. M2：增强演示表现（P1 完成，评分指标可展示）。
3. M3：答辩交付版（文档、视频、应急方案齐全）。

## 12. 风险与约束（非技术描述）

1. 现场演示时间短，必须降低操作复杂度。
2. 评审更关注“闭环价值与稳定性”，而非功能数量。
3. 多人协作时需统一演示口径，避免讲解不一致。

## 13. 成功定义

当评审可以在一次演示中清楚看到：

1. 数据持续被感知。
2. 控制命令有效执行。
3. 异常被及时发现与恢复。
4. 全过程可解释、可追溯、可复现。

则该产品达到课程设计目标。
